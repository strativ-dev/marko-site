The workflow

How it works, exactly.

The full workflow, step by step. No magic, no hand-waving.

1

Start a review

  • Press Alt+M on any website, or click the Marko toolbar icon then Start annotation.
  • The review toolbar appears top-center; a viewport-size pill (e.g. 1440 × 900) appears top-right.
  • The Pin tool is selected by default. Press ? anytime for the cheatsheet, Esc to exit.
2

Pin a comment to an element

  • Hover: the element under your cursor gets a green outline with a chip showing its tag, class, and size.
  • Click: a compact comment box opens with three tabs: Note, Suggestion, Guideline (Tab cycles them).
  • Nothing is saved yet: Submit stays disabled until you write something. Enter submits, Esc or Cancel discards, clicking elsewhere replaces the draft.
  • On submit the pin becomes a numbered dot anchored to that element. It stays attached through scrolling, window resizing, page changes, and reloads.
3

Manage the pin

  • Click a numbered dot: a sidebar docks to the left with the pin's author and time.
  • Set severity (Neutral to Critical), assignee (autocompletes from your team list), and due date.
  • Tag a guideline: Nielsen's 10 usability heuristics plus any custom ones you define in Settings.
  • Discuss in the comment thread with @mentions, emoji, and image attachments. Tick the checkbox to mark the pin resolved (its dot turns green ✓).
  • Navigate pins with ‹ N/M ›. Delete via the sidebar, or by clicking the dot with the eraser tool.
4

Annotate visually (optional)

  • Draw with arrow, box, pen, text, and a ruler that live-measures px and angle.
  • Select tool moves any shape and resizes via endpoint handles. Ctrl+Z / Ctrl+Shift+Z undo/redo.
  • Inspect tools: element inspector (box model, typography, colors, copy CSS/selector), font inspector, and a native eyedropper.
  • Everything autosaves per URL and restores when you come back.
5

Save a report

  • Click + Report: Marko captures the visible page with your annotations baked into the screenshot.
  • A dialog collects title, note, suggestion, severity, and active/resolved state. The project is auto-detected from the site (override or create a new one inline).
  • Saved locally with metadata: OS, browser, viewport size, pixel ratio, URL, timestamp, plus every pin's full detail.
6

Triage in the dashboard

  • Toolbar icon, then Reports. Projects live in a sidebar (Active/Archived, search, rename, archive).
  • Switch grid ⇄ kanban: drag cards between severity columns to re-triage. Tick a card's checkbox to resolve the whole report.
  • Filter by status, severity, device (mobile/tablet/desktop), assignee, due date, or created date range, or free-text search.
  • Open a report for tabs: Description · Pins · Comments · Activity · Meta. Every edit autosaves, and every event lands in the activity feed.
7

Share it

  • Standalone HTML: one self-contained file with screenshot, pins, and comments. Opens in any browser; recipients need nothing installed.
  • Share link: the report travels compressed inside the URL (works for other Marko users), or through your own backend for short team links (reference server included, optional per-link passwords).
  • Per-pin deep links that scroll straight to a pin, whole-project exports, PDF via print, CSV of reports or pins, and JSON backup/restore.
  • Invite teammates from the project sidebar. Marko drafts the email with the link or file attached.

Get started

Installed in two minutes.

Marko ships as an open-source package while the Chrome Web Store listing is in review.

1

Download the latest release zip and unzip it.

2

Open chrome://extensions in Chrome.

Paste it into the address bar. Chrome blocks direct links to it.

3

Turn on Developer mode (top-right toggle).

4

Click Load unpacked and select the unzipped folder.

5

Pin Marko to your toolbar, open any site, press Alt+M. Done.

Questions

Frequently asked, honestly answered.

Is Marko really free?

Yes. Free forever, for everyone. No trial, no pro tier, no seat licenses. It's MIT-licensed open source.

How do I share reports if everything is local?

Four ways: a self-contained HTML file that opens in any browser (no extension needed), a compressed share link for other Marko users, PDF/CSV exports, or, for teams that want real short links, point Marko at your own tiny backend (a reference server ships in the repo).

Do report recipients need Marko installed?

No. The standalone HTML export and PDF contain everything: screenshot, pins, comments, and metadata.

What data does Marko collect?

None. There is no server to send it to. Read the privacy policy, it's one page.

Which browsers are supported?

Chrome and Chromium-based browsers (Edge, Brave, Arc, Opera) with Manifest V3 support.

Can my team collaborate?

Yes. Share project links or files, assign pins to teammates, @mention them in comments, and track resolve states. For always-live team links, self-host the optional backend (one Node file, one dependency).
