The workflow

How it works, exactly.

The full workflow, step by step. No magic, no hand-waving.

1

Sign up once

  • Install the extension, click the toolbar icon, hit Get started.
  • Sign up on the web with email and password. The extension picks up your session automatically; no copy-pasting tokens.
  • From then on the extension and the web app share one account.
2

Create a project for a site

  • Open the site you're reviewing, click the Marko icon.
  • If a project already exists for this site, Marko shows it with its active and resolved counts.
  • Otherwise one click on Create project for this page starts one and drops you straight into review mode.
3

Pin comments on the live page

  • In comment mode (bottom-center toggle) click any element: a composer opens right there.
  • Write the comment, set a severity from Neutral to Critical, mention teammates with @.
  • The pin anchors to the element and survives scrolling and window resizing. Flip the toggle to browse the site normally.
4

Discuss and resolve in place

  • Click a pin to open its thread: reply, change severity, tick resolve.
  • The panel (bottom-left bubble) lists every heurio in Active and Resolved tabs; clicking one scrolls the page to its pin, navigating to the right URL if needed.
  • The top-right pill resizes your window to any device preset: 4K down to iPhone SE, so you can review every breakpoint.
5

Track everything in the web app

  • Open in Marko app takes you to the project dashboard at app.html.
  • Projects live in a sidebar; each project is a kanban board with a column per severity.
  • Filter by status, assignee, or free text. Open any card for the full thread, assignee, and due date. Resolve with one checkbox.
6

Bring the team

  • Share (in the dock or the web app) invites teammates by email; they sign up with their own account and see the same board.
  • Copy link puts the project URL on your clipboard for chat or tickets.
  • Everything is stored in the project backend, a tiny Cloudflare Worker. Use ours, or deploy your own from the source repo and point Marko at it in Settings.

Get started

Installed in two minutes.

Marko ships as an open-source package while the Chrome Web Store listing is in review.

1

Download the latest release zip and unzip it.

2

Open chrome://extensions in Chrome.

Paste it into the address bar. Chrome blocks direct links to it.

3

Turn on Developer mode (top-right toggle).

4

Click Load unpacked and select the unzipped folder.

5

Pin Marko to your toolbar, open any site, press Alt+M. Done.

Questions

Frequently asked, honestly answered.

Everything you need to know before installing, from privacy to how sharing actually works.

Is Marko really free?

Yes. Free forever, for everyone. No trial, no pro tier, no seat licenses. MIT-licensed open source.

Where is my data stored?

Projects and their comments live in the Marko backend, a small Cloudflare Worker with a KV store. Use the hosted one that ships with the extension, or deploy your own and own every byte. Draft annotations stay in Chrome's local storage on your device.

How does sharing work?

Invite teammates by email from the in-page dock or the web app; they sign up and see the same project board. Or copy the project link and drop it in chat. Standalone HTML, PDF, and CSV exports still exist for people outside the team.

Do recipients need Marko installed?

No. Standalone HTML and PDF exports contain everything: screenshot, pins, comments, and metadata. They open in any browser.

How is team sharing authenticated?

Email and password against the Marko backend, a Cloudflare Worker you can also self-host. Passwords are hashed on the worker with PBKDF2-SHA256 (100,000 iterations, per-user salt) and only the hash lives in your Workers KV. Sessions are 30-day HMAC tokens signed by a secret you set with wrangler secret put SIGNING_SECRET. No third-party identity provider, no plaintext passwords in storage, no shared team credentials.

Which browsers are supported?

Chrome and Chromium-based browsers with Manifest V3 support: Edge, Brave, Arc, Opera, and Vivaldi.

Can my team collaborate?

Yes. Share project links or files, assign pins to teammates, @mention them in comments, and track resolve states across a shared workspace.

Do I need my own Cloudflare account?

No. Marko ships with a hosted worker so you can sign up and start reporting straight away. If you'd rather run the backend yourself, deploy the worker in the cloudflare-worker folder of the source repo (5-minute guide) and paste the resulting URL into Marko Settings.
